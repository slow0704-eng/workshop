────────────────────────────────────────────────────────────────
1. 사업 개요
────────────────────────────────────────────────────────────────

■ 서비스명
  마음 워크숍 (가칭)

■ 서비스 형태
  워크숍 북 형태의 체험형 웹사이트 (모바일 우선 반응형)

■ 핵심 콘셉트
  5분 이내로 완료할 수 있는 자기계발·심리 워크숍을
  디지털 워크북 형태로 제공하는 무료 공공 서비스.
  QR코드 하나로 축제·행사 현장에서 즉시 체험 가능.

■ 슬로건
  "5분, 나를 만나는 시간"

■ 운영 모델
  공공 목적 무료 서비스 (사회적 가치 창출 중심)

■ 개발 원칙
  - JavaScript 없이 순수 HTML + CSS만으로 구현
  - QR 스캔 후 3초 안에 시작 가능한 초경량 구조
  - 파일 하나 = 워크숍 하나 (완전 독립 구조)
  - 결과 화면은 "캡처해서 공유하고 싶은" 수준의 디자인

■ 핵심 유통 채널 [v3 신설]
  - 지역 청년축제·행사 QR 부스 (1차 유입 채널)
  - SNS 결과 공유 → "나도 해보기" 바이럴 (2차 확산)
  - 기관 제휴 링크 (대학 상담실, 기업 HR, 청년센터)

2. 문제 정의 및 필요성
────────────────────────────────────────────────────────────────

■ 현황 및 문제
  - 직장인 83.6%가 번아웃을 경험 (2025 잡코리아 조사)
  - 대학생 우울감 경험 비율 지속 증가 추세
  - 심리상담 비용 부담 및 접근성 문제
  - 기존 자기계발 콘텐츠는 긴 호흡 → 바쁜 현대인에게 부적합

■ 페르소나에서 확인된 구체적 문제 [v3 신설]
  - 김하늘(23세): MBTI 수십 번 했지만 "그래서 뭐?" → 깊이 부재
  - 이준호(28세): 명상앱 3일 만에 삭제 → 수동적 체험 안 맞음
  - 최민수(26세): 상담 비용 엄두 못 냄 → 무료 자기탐색 도구 필요
  - 박소연(31세): 축제 프로그램 예산 부족 → 무료 디지털 솔루션 수요

■ 해결 방향
  - 5분 이내 마이크로 워크숍으로 심리적 접근성 확보
  - 무료 제공으로 경제적 장벽 제거
  - 워크북 형태의 능동적 참여로 실질적 변화 유도
  - QR 하나로 어디서든 즉시 접속 가능한 웹 기반 서비스
  - "하는" 체험 (쓰기·선택하기) → 수동적 콘텐츠와 차별화

3. 대상 고객 (타겟 사용자) [v3 전면 재구성]
────────────────────────────────────────────────────────────────

■ 핵심 타겟: 청년층 (18~34세)

  [1] 대학생·취준생 (18~28세) — 최우선 타겟
      페르소나: 김하늘(23세), 최민수(26세)
      - 진로 불안, 자존감 저하, 자기탐색 욕구
      - MBTI 테스트 피로감 → "의미 있는" 심리 콘텐츠 수요
      - SNS 공유에 적극적 → 바이럴 유도의 핵심 세대
      - 축제·캠퍼스 행사에서 QR로 자연 유입 가능
      - 경제적 여유 없음 → 무료 서비스 높은 가치

  [2] 청년 직장인 (25~34세) — 주요 타겟
      페르소나: 이준호(28세)
      - 번아웃, 직무 고민 → 짧은 자기관리 도구 필요
      - 점심시간·퇴근 후 5분 활용
      - 동료 간 카톡 공유로 확산
      - 실천 가이드를 원함 (결과만이 아닌 행동 제안)

■ 확장 타겟: 기관 담당자 (B2G/B2B)

  [3] 청년센터·행사 기획자
      페르소나: 박소연(31세)
      - 예산 제약 속 참여형 프로그램 필요
      - QR 부스 설치만으로 운영 가능 → 비용 제로
      - 참여자 SNS 공유 → 행사 홍보 효과
      - 성과 보고서 실적으로 활용

  [4] 기업 HR 담당자
      페르소나: 정유진(34세)
      - 사내 복지·ESG 보고용 정신건강 프로그램
      - 인트라넷 링크 공유만으로 배포 가능
      - 전문가 감수 여부가 의사결정 핵심 요인

■ 보조 타겟
  - 심리에 관심 있는 일반 성인 (35세 이상)
  - 학교 상담실·청소년센터 (교육 프로그램 연계)

4. 서비스 구성 [v3 수정]
────────────────────────────────────────────────────────────────

■ 워크숍 투트랙 구조 [v3 신설]

  ┌─────────────────────────────────────────────────────────┐
  │  트랙 A: 1분 미니 워크숍 (진입용)                       │
  │  ─────────────────────────────────────────────         │
  │  - 1~2분 이내 완료                                     │
  │  - 선택형 질문 3~5개 → 결과 카드                       │
  │  - 축제 현장·SNS 바이럴 최적화                         │
  │  - 바이럴 테스트와 동일한 진입장벽                      │
  │  - "풀 워크숍도 해보세요" → 트랙 B로 유도              │
  │  - 예: "30초 감정 체크", "나의 에너지 잔량은?"          │
  └─────────────────────────────────────────────────────────┘
  ┌─────────────────────────────────────────────────────────┐
  │  트랙 B: 5분 풀 워크숍 (핵심 가치)                      │
  │  ─────────────────────────────────────────────         │
  │  - 3~5분 완료, 4단계 구조                               │
  │  - 탐색 → 체험 → 성찰 → 결과 + 실천 가이드           │
  │  - 자기이해·변화를 위한 본격 워크숍                     │
  │  - 시리즈/챌린지 연결로 재방문 유도                     │
  │  - 예: "나의 강점 5가지 발견하기", "가치관 카드 정렬"   │
  └─────────────────────────────────────────────────────────┘

  ★ 투트랙 전략 근거 (페르소나 기반)
  - 김하늘: 축제에서 1분 미니 → 집에서 풀 워크숍으로 전환
  - 이준호: 점심시간 풀 워크숍 → 동료에게 미니 워크숍 공유
  - 최민수: 혼자 있을 때 풀 워크숍 → 깊은 자기탐색
  - B군(바이럴 테스트) 대비 진입장벽을 동일 수준으로 낮춤

■ 풀 워크숍 4단계 구조 (1회 3~5분)

  [STEP 1] 도입 (30초)
      - 주제 소개 및 오늘의 질문 제시
      - "약 3분 소요" 예상 시간 명시 [v3: 이탈 방지]
      - CSS 애니메이션 감성 비주얼
      - 구현: 앵커 링크(#step1)로 페이지 내 이동

  [STEP 2] 탐색 (1분)
      - 짧은 심리 개념 설명 (카드 형태)
      - 구현: <details><summary> 또는 :target 카드 전환

  [STEP 3] 체험 (1분 30초)
      - 선택형 질문: radio/checkbox (메인)
      - 자유 서술: <textarea> (선택사항) [v3: 필수 아님으로 변경]
      - CSS :checked 선택자로 실시간 시각 피드백
      - 구현: <form> + CSS (JS 불필요)

  [STEP 4] 결과 + 실천 가이드 (1분) [v3 확장]
      - 선택 결과에 따른 맞춤 메시지 (CSS :checked + 형제 선택자)
      - 결과 카드 (정사각형 디자인, 공유 최적화)
      - "이번 주에 시도할 수 있는 것" 실천 가이드 [v3 신설]
      - 결과 공유 영역 (이미지 저장 / SNS 링크 / 텍스트 복사)
      - 다른 워크숍 추천 링크
      - 위기 상담 안내 (하단 상시 노출) [v3 의무화]

■ HTML-only 인터랙션 기법

  기법                    용도                    예시
  ─────────────────────────────────────────────────────────────
  :target 선택자          단계별 화면 전환        #step1, #step2 앵커
  :checked + ~ 선택자     선택에 따른 결과 표시   라디오 버튼 심리 테스트
  <details><summary>      정보 접기/펼치기        심리 개념 카드 열기
  <input type="range">    슬라이더 입력           감정 강도 체크
  CSS 애니메이션          시각 효과               페이드인, 슬라이드
  <progress>              진행률 표시             워크숍 진행 단계 바
  CSS scroll-snap         카드 넘기기 효과        좌우 스와이프 탐색

5. 핵심 기능 [v3 수정]
────────────────────────────────────────────────────────────────

  [1] CSS-only 인터랙티브 워크북
      - :target, :checked, <details> 등 순수 CSS/HTML 기법
      - 모바일 터치 대응 (CSS 기본 지원)
      - JavaScript 완전 배제 → 브라우저 호환성 극대화
      - QR 스캔 후 1초 이내 로딩 [v3: 축제 현장 대응]

  [2] 결과물 공유 시스템 [v3 전면 재설계]

      ┌─ 공유 경로 1: 이미지 저장 ──────────────────────────┐
      │ Phase 1: 스크린샷 최적화 결과 화면 + 안내 문구      │
      │          "이 화면을 캡처해서 저장하세요"              │
      │ Phase 3: html2canvas JS 도입 → 이미지 자동 저장     │
      └─────────────────────────────────────────────────────┘
      ┌─ 공유 경로 2: SNS 링크 공유 ────────────────────────┐
      │ HTML <a> 태그 + URL scheme (JS 불필요)              │
      │ 트위터(X) / 페이스북 / 카카오톡 URL 공유            │
      │ OG 메타 태그로 미리보기 카드 자동 생성               │
      │ "나도 해보기" 링크 = 워크숍 URL 그 자체             │
      └─────────────────────────────────────────────────────┘
      ┌─ 공유 경로 3: 텍스트 복사 ──────────────────────────┐
      │ Phase 1: 결과 텍스트 선택 영역 안내                 │
      │ Phase 3: Clipboard API(JS) 도입 → 원클릭 복사      │
      └─────────────────────────────────────────────────────┘

  [3] 결과 카드 디자인 시스템 [v3 신설]

      원칙 1: 정사각형 비율 (1:1)
        → 인스타 피드·카톡 프로필·스토리 모두 최적
      원칙 2: 고대비 + 감성 색상
        → 카테고리별 색상 톤, 스크린샷해도 예쁜 디자인
      원칙 3: URL 워터마크
        → 하단에 서비스 URL 작게 표시 → 자연 유입
      원칙 4: 공유하고 싶은 문구
        → "당신의 에너지 잔량: 23% ☁" 같은 인용성 높은 표현

  [4] QR/행사 대응 기능 [v3 신설]

      - 축제 전용 랜딩 페이지 (QR 연결용)
        → 행사명 + 추천 워크숍 큐레이션
      - 기관 공동 브랜딩 페이지
        → HTML 파일 1개 추가 = 커스텀 랜딩 완성
      - 초경량 HTML → 야외 불안정 네트워크에서도 즉시 로딩
      - 동시 접속 무제한 (정적 파일, CDN 서빙)

  [5] 위기 안전망 [v3 신설 — 의무 기능]

      - 모든 워크숍 하단에 위기 상담 안내 상시 노출
        ┌──────────────────────────────────────────────┐
        │ 지금 힘든 순간이라면, 혼자 감당하지 않아도 됩니다. │
        │ 자살예방상담전화 1393 | 정신건강위기상담 1577-0199  │
        │ 청소년상담 1388 | 카카오톡 플러스친구 #마음이음     │
        └──────────────────────────────────────────────┘
      - 자존감·감정 관련 워크숍에 "전문 상담 안내" 링크 추가
      - 부정적 결과에도 반드시 희망적 메시지 + 작은 실천 제안

  [6] 초경량 운영 설계
      - 서버·DB·JS 없이 순수 HTML+CSS만으로 운영
      - 콘텐츠 추가 = HTML 파일 1개 복사 + 내용 교체
      - 유지보수 업무 완전 제로 수준
      - 장애 가능성 사실상 없음 (정적 파일만 서빙)

6. 수익 모델 및 지속가능성 [v3 수정]
────────────────────────────────────────────────────────────────

■ 기본 원칙
  - 사용자에게 무료 제공 (공공 서비스 지향)
