페르소나 1: 김하늘 — "축제에서 우연히 만난 마음 워크숍"
────────────────────────────────────────────────────────────────

■ 기본 정보
  이름: 김하늘 (가명)
  나이: 23세
  직업: 대학교 3학년 (심리학 부전공)
  거주: 경기도 수원
  성격: 호기심 많고 새로운 경험에 열린 편
        SNS에 일상을 자주 공유함
        자기탐색에 관심 있지만 체계적으로 해본 적 없음

■ 디지털 환경
  주 사용 기기: 스마트폰 (아이폰)
  주 SNS: 인스타그램 스토리, 카카오톡
  웹 사용 습관: 링크 클릭 → 모바일 브라우저에서 바로 소비
                앱 설치는 귀찮아서 웹 선호
  콘텐츠 소비: 숏폼 영상, 심리테스트, 카드뉴스

■ 심리·정서 상태
  - 전공 선택 고민, 졸업 후 진로 불안
  - "나는 뭘 좋아하는 걸까"라는 막연한 자기탐색 욕구
  - MBTI 테스트는 수십 번 해봤지만 "그래서 뭐?" 라는 느낌
  - 전문 상담은 아직 필요성을 못 느낌 (비용도 부담)

■ 유입 시나리오
  ┌─────────────────────────────────────────────────────────────┐
  │ 상황: 지역 청년축제에 친구와 놀러 옴                        │
  │                                                             │
  │ 1. 축제 부스에서 "5분 마음 워크숍" QR코드 발견              │
  │    → "심리테스트인가?" 하고 스마트폰으로 QR 스캔            │
  │                                                             │
  │ 2. 모바일 브라우저에서 바로 열림 (앱 설치 X, 로그인 X)      │
  │    → "오 깔끔하다, 바로 되네"                               │
  │                                                             │
  │ 3. "오늘의 감정 날씨 체크" 워크숍 시작                      │
  │    → 질문에 답하고 자기 감정을 선택하는 과정이 새로움       │
  │    → "단순 테스트가 아니라 생각하게 되네"                   │
  │                                                             │
  │ 4. 결과 화면 → 스크린샷 캡처 / 이미지 저장                 │
  │    → 인스타 스토리에 올림 "축제에서 한 마음 워크숍 괜찮다"  │
  │    → 친구가 "나도 해볼래" → QR 다시 스캔                   │
  │                                                             │
  │ 5. 집에 돌아와서 다른 워크숍도 해봄                         │
  │    → "가치관 카드 정렬하기" → 진로 고민에 도움              │
  │    → 7일 챌린지 시작                                       │
  └─────────────────────────────────────────────────────────────┘

■ 니즈 & 기대
  - 빠르게 시작, 빠르게 끝나야 함 (축제 현장에서 5분 이내)
  - 앱 설치·회원가입 절대 하기 싫음
  - 결과가 "예쁘게" 나와야 공유하고 싶음
  - 단순 유형 분류가 아닌, 뭔가 생각할 거리가 있으면 좋겠음
  - 친구와 결과 비교하는 재미

■ 불만 포인트 (Pain Points)
  - 기존 심리테스트: "또 MBTI야? 이제 지겨워"
  - 광고 떡칠된 테스트 사이트: 결과 보기 전에 전면광고 → 짜증
  - 결과가 안 예쁘면 공유 안 함
  - 네트워크 느리면 축제 현장에서 로딩 안 됨 → 포기

■ 마음 워크숍이 충족해야 할 것

  니즈                    마음 워크숍의 대응
  ─────────────────────────────────────────────────────────────
  즉시 접속               QR → 모바일 브라우저 바로 로딩
                          HTML-only → 초경량, 느린 네트워크 OK
  설치·가입 없음          정적 HTML, 데이터 수집 제로
  5분 이내 완료           워크숍 구조 자체가 5분 설계
  예쁜 결과 화면          정사각형 결과 카드 디자인
                          스크린샷/이미지 저장 최적화
  공유 편의성             결과 화면 이미지 저장 버튼
                          SNS 공유 URL 링크
  깊이 있는 체험          단순 분류 X → 자기성찰 유도 구조
  재방문 동기             시리즈·챌린지 안내

페르소나 2: 이준호 — "점심시간에 잠깐 쉬고 싶은 직장인"
────────────────────────────────────────────────────────────────

■ 기본 정보
  이름: 이준호 (가명)
  나이: 28세
  직업: IT 스타트업 주니어 개발자 (입사 2년차)
  거주: 서울 성수동
  성격: 내향적이지만 자기계발에 관심 많음
        혼자 조용히 하는 활동 선호
        SNS는 잘 안 하지만 괜찮은 건 카톡으로 공유

■ 디지털 환경
  주 사용 기기: 스마트폰 (안드로이드) + 업무용 노트북
  주 SNS: 카카오톡 (개인/그룹), 가끔 트위터(X)
  웹 사용 습관: 점심시간에 스마트폰으로 콘텐츠 소비
                기술 블로그, 뉴스레터 구독
  콘텐츠 소비: 긴 글 읽기 가능, 질 좋은 콘텐츠 선호

■ 심리·정서 상태
  - 번아웃 초기 증상 (의욕 저하, 주말에도 피곤)
  - "이 회사 계속 다녀야 하나" 고민
  - 전문 상담까지는 아닌 것 같고, 혼자 정리하고 싶음
  - 명상 앱 깔아봤지만 3일 만에 삭제함 (안 맞음)

■ 유입 시나리오
  ┌─────────────────────────────────────────────────────────────┐
  │ 상황: 회사 동료가 카카오톡으로 링크 공유                     │
  │ "이거 해봐, 번아웃 자가진단인데 좀 괜찮아"                  │
  │                                                             │
  │ 1. 점심시간에 링크 클릭 → 모바일 브라우저 오픈              │
  │    → 깔끔한 디자인, 광고 없음 → "오 괜찮네"                │
  │                                                             │
  │ 2. "스트레스 체크: 나의 에너지 잔량은?" 워크숍 시작          │
  │    → 질문에 답하면서 자기 상태를 객관적으로 보게 됨          │
  │    → "아, 내가 꽤 지쳐 있구나" 인지                         │
  │                                                             │
  │ 3. 결과 화면 → 실천 가이드 포함                             │
  │    → "이번 주에 시도할 수 있는 3가지" 구체적 제안            │
  │    → 결과 이미지 저장 → 폰 갤러리에 보관                   │
  │                                                             │
  │ 4. 동료에게 카톡 답장 "이거 좋다 ㅋㅋ 나 에너지 23%래"      │
  │    → 동료: "ㅋㅋㅋ 나 17%ㅠ"                               │
  │    → 팀 단톡방에 공유 → 팀원들도 해봄                      │
  │                                                             │
  │ 5. 다음 날 퇴근 후 "업무 우선순위 매트릭스" 워크숍 해봄      │
  │    → 업무 정리에 실제로 도움 → 북마크 저장                  │
  └─────────────────────────────────────────────────────────────┘

■ 니즈 & 기대
  - 점심시간 10분 안에 시작-완료 가능해야 함
  - 유치하지 않은 톤 (성인 직장인 대상 느낌)
  - 결과가 실질적으로 도움이 되면 좋겠음 (실천 가이드)
  - 결과를 저장해뒀다가 나중에 다시 보고 싶음
  - 광고 없이 깨끗한 경험

■ 불만 포인트 (Pain Points)
  - 명상 앱: "나는 가만히 듣는 게 안 맞아"
  - 자기계발 강의: 30분씩 볼 시간 없음
  - 심리테스트: 유치하거나 광고 범벅
  - 심리상담: "아직 그 정도는 아닌 것 같은데..."

■ 마음 워크숍이 충족해야 할 것

  니즈                    마음 워크숍의 대응

  짧은 시간에 완료        5분 이내 설계
  성숙한 톤앤매너         성인 타겟 디자인·문체
  실질적 도움             결과 + 실천 가이드 제공
  결과 보관               이미지 파일 저장 기능
                          캡처 유도 디자인
  깨끗한 경험             광고 제로, 공공 서비스
  능동적 참여             쓰기·선택하기 중심 (듣기 X)

페르소나 3: 박소연 — "축제 기획하면서 프로그램 찾는 담당자"
────────────────────────────────────────────────────────────────

■ 기본 정보
  이름: 박소연 (가명)
  나이: 31세
  직업: 지자체 청년센터 담당자 (계약직 3년차)
  거주: 대전 유성구
  성격: 꼼꼼하고 실행력 있음
        새로운 프로그램 발굴에 적극적
        예산 제약 하에서 최대 효과를 내야 하는 입장

■ 디지털 환경
  주 사용 기기: 업무용 PC + 스마트폰
  주 SNS: 인스타그램 (센터 공식 계정 운영)
  웹 사용 습관: 벤치마킹용 서비스 탐색을 자주 함
  콘텐츠 소비: 공공기관 사례집, 청년 정책 뉴스

■ 심리·정서 상태
  - 본인도 청년 세대로서 공감하는 부분 많음
  - "올해 축제에 뭔가 새로운 프로그램 넣고 싶다"
  - 예산은 적은데 참여형 콘텐츠를 원하는 상황
  - 디지털 프로그램이면 인쇄물 비용도 절감 가능

■ 유입 시나리오
  ┌─────────────────────────────────────────────────────────────┐
  │ 상황: 청년축제 프로그램 기획 중 웹 검색                      │
  │ "청년 정신건강 프로그램 무료"로 검색                         │
  │                                                             │
  │ 1. 마음 워크숍 사이트 발견                                   │
  │    → 무료·공공 서비스 → "예산 안 들겠는데?"                 │
  │    → 직접 워크숍 3~4개 해봄 → "이거 괜찮다"                │
  │                                                             │
  │ 2. 축제 부스 연계 가능성 검토                                │
  │    → QR코드만 인쇄하면 됨 → 비용 최소                      │
  │    → 앱 설치 불필요 → 현장 참여 장벽 낮음                  │
  │    → 참여자가 결과를 SNS에 공유 → 축제 홍보 효과            │
  │                                                             │
  │ 3. 마음 워크숍 운영자에게 협업 제안                          │
  │    → "청년축제에서 QR 부스로 운영하고 싶습니다"             │
  │    → 축제 특별 워크숍 공동 기획 가능성 논의                 │
  │    → 센터 로고 + 마음 워크숍 공동 브랜딩                    │
  │                                                             │
  │ 4. 축제 당일                                                 │
  │    → QR 스탠드 설치 + 안내 포스터                           │
  │    → 참여자 100명+ → SNS 공유 활발                         │
  │    → 축제 성과 보고서에 "디지털 정신건강 프로그램" 기재     │
  └─────────────────────────────────────────────────────────────┘

■ 니즈 & 기대
  - 무료여야 함 (예산 제약)
  - QR코드 하나로 현장에서 바로 작동해야 함
  - 참여자가 별도 설치·가입 없이 즉시 가능해야 함
  - 사후 보고서에 실적으로 넣을 수 있는 공신력
  - 참여자 SNS 공유 → 축제 자체 홍보 효과

■ 마음 워크숍이 충족해야 할 것

  니즈                    마음 워크숍의 대응
  ─────────────────────────────────────────────────────────────
  무료 사용               공공 서비스, 비용 제로
  현장 즉시 접속          QR → HTML 즉시 로딩, 앱 X
  설치·가입 없음          정적 HTML, 데이터 수집 없음
  공신력                  전문가 감수, 공공 브랜딩
  홍보 효과               참여자 SNS 공유 → 자연 홍보
  커스터마이징            축제 특별 워크숍 공동 기획 가능
                          (HTML 파일 1개 추가 = 특별 워크숍)

페르소나 4: 최민수 — "고시원에서 혼자 공부하는 취준생"
────────────────────────────────────────────────────────────────

■ 요약 프로필
  나이: 26세 / 취업준비생 (졸업 1년차)
  거주: 서울 관악구 고시원
  상태: 취업 실패 반복으로 자존감 저하, 고립감
        상담 받고 싶지만 비용 엄두 못 냄
        밤에 혼자 있을 때 불안감이 심해짐

■ 유입 경로
  - 에브리타임 게시판에서 누군가 공유한 링크
  - "자존감 테스트인데 광고 없고 괜찮음" 이라는 댓글

  - 위기 상담 안내 (자살예방 상담전화 등) 상시 노출
  - 공감적이고 따뜻한 톤의 결과 메시지
  - "지금 힘들다면" → 전문기관 연계 링크 제공
  - 부정적 결과에도 희망적 메시지 + 실천 가능한 작은 제안

페르소나 5: 정유진 — "사내 복지 프로그램 고민하는 HR 담당자"
────────────────────────────────────────────────────────────────

■ 요약 프로필
  나이: 34세 / 중견기업 인사팀 대리
  상태: ESG 보고서에 넣을 임직원 정신건강 프로그램 필요
        예산 확보는 가능하지만 ROI 설명이 필요
        기존 EAP(근로자 지원 프로그램) 이용률 낮음

■ 유입 경로
  - "직장인 정신건강 프로그램 무료"로 검색
  - 또는 청년축제에서 본 후 회사 적용 검토

■ 핵심 니즈
  - 임직원이 쉽게 참여할 수 있는 프로그램 (설치·가입 X)
  - 회사 인트라넷에 링크만 걸면 되는 구조
  - 참여 실적을 ESG 보고서에 기재할 수 있는 근거
  - 전문성 있는 콘텐츠 (전문가 감수 여부 중요)

■ 마음 워크숍이 충족해야 할 것
  - 기업 전용 랜딩 페이지 또는 공동 브랜딩 가능성
  - 전문가 감수 인증 명시
  - 사용 실적 리포트 제공 방안 (구글 폼 등 간접 측정)

────────────────────────────────────────────────────────────────
10. 페르소나에서 도출한 핵심 설계 원칙
────────────────────────────────────────────────────────────────

  [1] "QR 찍고 3초 안에 시작"
      - 모든 페르소나의 공통 기대: 즉시 접속, 즉시 시작
      - HTML-only 초경량 구조가 이를 완벽히 충족
      - 앱 설치·회원가입은 절대적 이탈 요인

  [2] "결과가 예뻐야 공유한다"
      - 결과 카드 디자인 = 바이럴의 핵심 동력
      - 기능(이미지 자동 생성)보다 디자인(스크린샷 품질)에 투자
      - 정사각형, 감성 색상, 공유하고 싶은 문구

  [3] "재미있지만 의미 있어야 한다"
      - 바이럴 테스트 수준의 가벼움 + 전문 워크숍의 깊이
      - "해봤는데 좋았다"가 아닌 "해봤는데 생각할 게 생겼다"
      - 이것이 경쟁자와의 본질적 차별점

  [4] "축제 현장에서 작동해야 한다"
      - 불안정한 네트워크, 다양한 기기, 짧은 집중 시간
      - HTML-only의 기술적 제약이 오히려 최적의 해답
      - QR 포스터 1장 = 무한 참여자 수용 가능

  [5] "혼자 있는 사람에게도 따뜻해야 한다"
      - 축제의 활기찬 맥락뿐 아니라
        고시원에서 혼자 하는 사용자도 존재
      - 위기 상담 안내 상시 노출은 선택이 아닌 의무
      - 공감적 톤과 희망적 메시지 설계
