작성일: 2026년 3월 14일
기준 문서: 단위테스트시나리오 v1.0 / 유스케이스명세서 v1.0

문서 목적:
  단위 테스트가 "개별 컴포넌트가 작동하는가"를 검증한다면,
  통합 테스트는 "사용자가 시작부터 끝까지 막힘 없이
  경험을 완료할 수 있는가"를 검증한다.

  페르소나 시나리오 기반으로, 실제 사용 흐름 전체를
  연속으로 수행하며 화면 간 연계·데이터 흐름·감성 경험을
  종합적으로 검증.

테스트 ID 규칙:
  IT-[시나리오 번호]-[단계 번호]

테스트 환경:
  1차: Chrome DevTools Device Mode (375px, iPhone SE)
  2차: 실기기 (가용 시)

IT-01: 축제 QR → 미니 완주 → 공유 → 재방문 → 풀 전환
────────────────────────────────────────────────────────────────

  ■ 시나리오 배경

  페르소나:    김하늘 (23세, 대학생)
  상황:        지역 청년축제에 친구와 놀러 옴
               축제 부스에서 QR 포스터 발견
  환경:        야외, LTE 불안정, iPhone SE
  목적:        첫 방문 → 미니 체험 → 공유 → 재방문 전환 흐름 검증

  ■ 테스트 단계

  IT-01-01  QR/URL 접속
  ──────────────────────────────────────────────────────────
  행동:     index.html URL 접속 (QR 시뮬레이션)
  검증:     □ 1초 이내 로딩 (체감)
            □ 히어로 영역 + 캐릭터 표시
            □ 미니 카드 2개 + 풀 리스트 3개 표시
            □ 앱 설치·로그인 요구 없음

  IT-01-02  미니 워크숍 선택
  ──────────────────────────────────────────────────────────
  행동:     "나의 에너지 잔량은?" 카드 탭
  검증:     □ 에너지잔량.html 로딩
            □ 시작 화면 표시 (캐릭터 bounce)
            □ "같이 해보자!" 버튼 표시
            □ "답은 저장되지 않아요" 안내 표시

  IT-01-03  시작 → Q1 전환
  ──────────────────────────────────────────────────────────
  행동:     "같이 해보자! →" 탭
  검증:     □ Q1 화면 표시 (fadeIn)
            □ 시작 화면 완전 숨김 ★핵심
            □ "같이 해보자" 버튼 보이지 않음 ★핵심
            □ 진행률 바 1/4 표시
            □ 캐릭터(경청) + 안내 대사

  IT-01-04  Q1 선택 + 캐릭터 반응
  ──────────────────────────────────────────────────────────
  행동:     "🌧 일어나기 싫었다" 선택
  검증:     □ 선택지 스타일 변경 (배경+보더+체크)
            □ 캐릭터 반응 fadeIn: "좀 피곤한 아침이었나 봐."
            □ 다른 선택지는 미선택 상태

  IT-01-05  Q1 → Q2 → Q3 → Q4 순차 진행
  ──────────────────────────────────────────────────────────
  행동:     각 질문에서 선택 → "다음 →" 반복
  검증:     □ 각 단계 전환 시 이전 단계 완전 숨김
            □ 진행률 바 25→50→75→100% 정확
            □ 모든 단계에서 버튼 1개만 표시 (혼동 없음)

  IT-01-06  결과 확인
  ──────────────────────────────────────────────────────────
  행동:     Q4에서 "🪫 전혀 없다" 선택 → "결과 보기 →"
  검증:     □ 결과 카드 slideUp 등장
            □ 23% ☁ 표시 (부정 결과)
            □ 정사각형 카드, 카테고리 색상(보라)
            □ 캐릭터(위로): 검열자 대응 메시지
            □ "당신 잘못이 아니야" 포함
            □ 전문 상담 안내 추가 표시
            □ 워터마크 URL 표시

  IT-01-07  결과 공유
  ──────────────────────────────────────────────────────────
  행동:     "링크 공유하기" → "공유 링크 열기" → 트위터 탭
  검증:     □ 공유 패널 펼침 (SNS 3개 + URL)
            □ 트위터 앱/웹 열림
            □ 공유 텍스트에 워크숍 URL 포함
            □ "안 해도 괜찮아" 안내 표시

  IT-01-08  결과 → 풀 워크숍 전환
  ──────────────────────────────────────────────────────────
  행동:     추천 "오늘의 감정 날씨 체크" 탭
  검증:     □ 감정날씨.html 로딩
            □ 도입 화면 정상 표시
            □ 이전 워크숍 상태 잔존 없음 (새 페이지)

  IT-01-09  풀 워크숍 도입 → 탐색 전환
  ──────────────────────────────────────────────────────────
  행동:     "같이 해보자!" → 탐색 화면
  검증:     □ <details> 카드 4장 표시
            □ 카드 탭 시 내용 펼침/접힘
            □ 캐릭터: "천천히 살펴봐. 정답은 없어."

  IT-01-10  풀 워크숍 체험 → 결과
  ──────────────────────────────────────────────────────────
  행동:     체험에서 "☁ 흐림" 선택 → 서술 건너뜀 → 결과
  검증:     □ 서술 안 해도 결과 정상 표시
            □ 흐림 결과 카드 표시
            □ 캐릭터 위로 대사
            □ 실천 가이드 3가지 + "제안이야" 톤

  ■ 종합 판정

  전체 흐름 소요 시간: ___분 (목표: 미니 1분 + 풀 5분 이내)
  이탈 발생 지점: _________________ (없으면 "없음")
  버튼 혼동 발생: □ 없음  □ 있음 (위치: _______)
  감성 경험 평가: □ 자연스러움  □ 어색함 (구간: _______)
  판정: □ Pass  □ Fail

IT-02: 카톡 링크 → 풀 완주 → 결과 저장 → 동료 공유
────────────────────────────────────────────────────────────────

  ■ 시나리오 배경

  페르소나:    이준호 (28세, 직장인)
  상황:        점심시간, 동료가 카톡으로 에너지 잔량 링크 공유
  환경:        사내 Wi-Fi, Android Chrome
  목적:        카톡 유입 → 풀 체험(서술 포함) → 저장·공유 흐름 검증

  ■ 테스트 단계

  IT-02-01  카톡 링크 접속
  ──────────────────────────────────────────────────────────
  행동:     mini/에너지잔량.html URL 직접 접속
  검증:     □ 시작 화면 즉시 표시
            □ 캐릭터 안내 대사 표시
            □ "같이 해보자!" 버튼만 표시

  IT-02-02  미니 완주 → 풀 전환
  ──────────────────────────────────────────────────────────
  행동:     Q1~Q4 선택 → 결과 확인 → 추천 "감정 날씨" 탭
  검증:     □ 미니 전 과정 막힘 없음
            □ 추천 링크 정상 이동

  IT-02-03  풀 워크숍 전 과정 (서술 포함)
  ──────────────────────────────────────────────────────────
  행동:     도입 → 탐색(카드 2장 펼침) → 체험(선택+서술 입력) → 결과
  검증:     □ 탐색 카드 펼침/접힘 정상
            □ 감정 선택 후 캐릭터 반응
            □ 슬라이더 터치 조작 가능
            □ textarea에 한 줄 입력
            □ 입력 후 "결과 보기" 정상 작동
            □ 결과 카드 표시 (선택에 따른 올바른 분기)

  IT-02-04  결과 이미지 저장
  ──────────────────────────────────────────────────────────
  행동:     결과 카드 스크린샷 촬영
  검증:     □ 결과 카드가 한 화면에 보임
            □ 스크린샷에 카드 전체 + 워터마크 포함
            □ 카테고리 색상 정확
            □ 캐릭터 포함

  IT-02-05  동료에게 카톡 공유
  ──────────────────────────────────────────────────────────
  행동:     공유 URL 복사 → 카톡 대화방에 붙여넣기
  검증:     □ URL 텍스트 전체 선택 (user-select: all)
            □ 카톡에서 OG 미리보기 표시 (배포 후)
            □ 미리보기에 제목 + 설명 포함

  IT-02-06  결과에서 홈 복귀
  ──────────────────────────────────────────────────────────
  행동:     헤더 "홈" 탭
  검증:     □ index.html 이동
            □ 이전 워크숍 상태 잔존 없음
            □ 메인 페이지 정상 표시

  ■ 종합 판정

  전체 흐름 소요 시간: ___분
  서술 입력 부담감: □ 없음  □ 약간  □ 부담됨
  공유 경로 발견 용이성: □ 바로 찾음  □ 헤맴 (_____)
  판정: □ Pass  □ Fail

IT-03: 에브리타임 → 부정 결과 → 위기 안내 → 다른 워크숍
────────────────────────────────────────────────────────────────

  ■ 시나리오 배경

  페르소나:    최민수 (26세, 취준생)
  상황:        밤에 혼자, 에브리타임에서 링크 발견
  환경:        고시원 Wi-Fi, 스마트폰
  목적:        부정 결과 → 위기 안전망 → 공감 메시지 연속 검증
               ★ 가장 취약한 사용자 시나리오

  ■ 테스트 단계

  IT-03-01  에브리타임 링크 접속
  ──────────────────────────────────────────────────────────
  행동:     mini/감정체크.html URL 접속
  검증:     □ 시작 화면 표시
            □ "어떤 감정이든 괜찮아" Permission 메시지
            □ "답은 저장되지 않아요" 안전 안내

  IT-03-02  부정적 감정 선택
  ──────────────────────────────────────────────────────────
  행동:     Q1 "😮‍💨 지쳐있다" 선택
  검증:     □ 캐릭터(공감): "많이 힘든 하루인가 봐."
            □ 비난·판단 표현 없음
            □ 공감적 톤 확인

  IT-03-03  전 과정 부정 선택으로 결과 도달
  ──────────────────────────────────────────────────────────
  행동:     Q2(슬라이더 최대) → Q3 "무채색" 선택 → 결과
  검증:     □ 🌧 비 결과 카드 표시
            □ 결과 문구: "좀 힘든 날이구나. 오늘은 나한테
              좀 더 너그러워져봐."
            □ 캐릭터(위로): 검열자 대응 메시지
              "여기까지 온 것만으로도 나를 돌보고 있는 거야."
            □ 전문 상담 안내 추가:
              "더 깊은 이야기를 나누고 싶다면,
               전문 상담사와 대화해보는 것도 좋아."

  IT-03-04  위기 상담 안내 접근성
  ──────────────────────────────────────────────────────────
  행동:     페이지 하단 스크롤
  검증:     □ 위기 안내 영역 표시 (캐릭터 미등장)
            □ 1393 전화 링크 탭 → 전화 앱 열림
            □ "혼자 감당하지 않아도 됩니다" 존댓말
            □ 위기 안내에서 부정 결과 상담 안내까지
              자연스럽게 연결되는 흐름

  IT-03-05  다른 워크숍으로 전환 (회복 시작)
  ──────────────────────────────────────────────────────────
  행동:     추천 "나의 에너지 잔량은?" 탭
  검증:     □ 에너지잔량.html 이동
            □ 이전 감정체크 상태 잔존 없음
            □ 새 워크숍 정상 시작 가능

  IT-03-06  두 번째 워크숍에서도 위기 안내 확인
  ──────────────────────────────────────────────────────────
  행동:     에너지잔량 페이지 하단 확인
  검증:     □ 위기 안내 동일하게 존재
            □ 전화 링크 정상 작동

  ■ 종합 판정

  ★ 이 시나리오의 핵심 검증 포인트:
  위기 안내 도달성: □ 자연스럽게 눈에 들어옴  □ 찾기 어려움
  부정 결과 톤: □ 공감적·위로적  □ 비난적·판단적 느낌
  전문 상담 안내 톤: □ 자연스러움  □ 강제적 느낌

IT-04: 메인 → 유형 C(골라쓰기) → 완주 → 공유
────────────────────────────────────────────────────────────────

  ■ 시나리오 배경

  페르소나:    윤서현 (29세, AW 경험자)
  상황:        인스타에서 결과 카드 보고 링크 클릭
  환경:        집, Wi-Fi, iPad
  목적:        유형 C(직접쓰기/골라쓰기 탭 전환) 전체 흐름 검증

  ■ 테스트 단계

  IT-04-01  메인 → 5년 후 엽서 선택
  ──────────────────────────────────────────────────────────
  행동:     index.html → "5년 후 나에게 보내는 엽서" 탭
  검증:     □ 5년후엽서.html 로딩
            □ 도입 화면 정상 (category-career 주황 테마)
            □ "뭘 써야 할지 모르겠으면 골라 써도 괜찮아"

  IT-04-02  탐색 카드 확인
  ──────────────────────────────────────────────────────────
  행동:     도입 → 탐색 → 카드 4장 펼침
  검증:     □ 4장 details 카드 정상 펼침/접힘
            □ 캐릭터: "편하게 읽어봐"

  IT-04-03  카드 2×2 선택
  ──────────────────────────────────────────────────────────
  행동:     체험 → "❤ 관계·사랑" 카드 탭
  검증:     □ 해당 카드 보더+그림자+체크 강조
            □ 캐릭터 반응: "사람과의 연결을 소중히 여기는구나."
            □ 다른 카드 비강조 상태

  IT-04-04  골라 쓰기 탭 (기본 활성)
  ──────────────────────────────────────────────────────────
  행동:     하단 스크롤 → 탭 영역 확인
  검증:     □ "☑ 골라 쓰기" 탭 기본 활성
            □ 롤모델 메시지 5개 선택지 표시
            □ textarea 숨김 상태

  IT-04-05  골라 쓰기 선택
  ──────────────────────────────────────────────────────────

  검증:     □ 선택됨 스타일 적용

  ──────────────────────────────────────────────────────────
  행동:     "✍ 직접 쓰기" 탭 탭
  검증:     □ textarea 표시
            □ 골라쓰기 선택지 숨김
            □ 캐릭터(경청): "자유롭게 적어봐..."
            □ "여기 적은 건 아무도 안 봐"

  ──────────────────────────────────────────────────────────

  검증:     □ 이전 페이지 textarea 내용 잔존 없음

IT-10: 콘텐츠 톤앤매너 E2E
────────────────────────────────────────────────────────────────

  ■ 목적: 전체 흐름에서 아티스트 웨이 기반 톤앤매너 일관성 검증

  ■ 검증 체크리스트 (전 워크숍 공통)

  [Permission 메시지]
  □ 모든 도입 화면에 "~해도 괜찮아" 류 메시지 존재
  □ 캐릭터 대사로 전달 (반말+따뜻한 톤)

  [안전 공간]
  □ 모든 도입에 "답은 저장되지 않아요" 존재
  □ 서술 영역에 "여기 적은 건 아무도 안 봐" 존재

  [캐릭터 톤]
  □ 모든 대사가 반말 (존댓말 혼용 없음)
  □ 판단·비난 표현 없음
  □ 강제 표현 없음 ("반드시", "해야" 등)

  [결과 톤]
  □ 부정 결과에 "당신 잘못이 아니야" 류 포함
  □ 모든 결과에 희망적·공감적 메시지
  □ 실천 가이드 "이건 그냥 제안이야" 톤

  [공유 톤]
  □ "안 해도 괜찮아" 포함
  □ 공유를 강제하는 표현 없음

  [위기 안내 톤]
  □ 존댓말 사용 (캐릭터 반말과 구분)
  □ 캐릭터 미등장
  □ 진지하되 따뜻한 톤

  ■ 판정: □ 전체 일관  □ 불일치 발견 (위치: _______)
