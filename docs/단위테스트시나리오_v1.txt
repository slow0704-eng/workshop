================================================================
        "마음 워크숍" 단위 테스트 시나리오
        v1.0
================================================================

작성일: 2026년 3월 14일
기준 문서: 요구사항정의서 v1.0 / UI설계서 v3.0
          품질보증계획서 v1.0 / 인터페이스명세서 v1.0

문서 목적:
  구현된 모든 UI 컴포넌트·인터랙션·화면 전환을 단위 수준에서
  검증하는 테스트 케이스 정의. 1인 개발자가 배포 전 수동으로
  수행할 수 있는 체크리스트 형식.

테스트 ID 규칙:
  TC-[영역]-[번호]
  영역: NAV(접속·탐색), TRN(단계 전환), SEL(선택지),
        CHR(캐릭터), RES(결과), WRT(서술), SHR(공유),
        CRS(위기 안내), STY(스타일·레이아웃),
        PRF(성능), PRI(프라이버시), ACC(접근성)

판정 기준:
  P = Pass (통과)
  F = Fail (실패)
  N/A = 해당 없음

테스트 환경:
  기기 1: iPhone SE (375px) — iOS Safari
  기기 2: Galaxy S24 (360px) — Android Chrome
  기기 3: PC Chrome (1920px) — 최신 버전
