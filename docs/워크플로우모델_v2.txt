────────────────────────────────────────────────────────────────
2. 안전감 설계 — 보이지 않지만 모든 곳에 있는 구조
────────────────────────────────────────────────────────────────

  ★ 아티스트 웨이 핵심 통찰:
  "사람들이 자기탐색을 하지 못하는 가장 큰 이유는
   기능이 없어서가 아니라 안전하지 않다고 느끼기 때문이다."

  ■ 안전감 3계층

  ┌─────────────────────────────────────────────────────────────┐
  │ 계층 1: 기술적 안전 (사용자가 확인 가능)                     │
  │ ─────────────────────────────────────────────               │
  │ · 데이터 수집 없음 — "당신의 답은 저장되지 않습니다"         │
  │ · 광고 없음 — 제3자에게 노출될 걱정 없음                    │
  │ · 회원가입 없음 — 신원이 드러날 걱정 없음                   │
  │ · 새로고침 시 초기화 — 흔적이 남지 않음                     │
  │ → 캐머런의 "아무도 읽지 않는 모닝 페이지"의 디지털 구현     │
  ├─────────────────────────────────────────────────────────────┤
  │ 계층 2: 정서적 안전 (톤앤매너로 구현)                        │
  │ ─────────────────────────────────────────────               │
  │ · 도입: "지금 느끼는 감정, 뭐든 괜찮아요"                   │
  │ · 체험: "(선택) 적어보세요" — 강제 없는 초대                │
  │ · 결과: "이 워크숍을 해본 것 자체가 의미 있어요"            │
  │ · 판단 없음 — 어떤 결과도 "좋다/나쁘다" 평가 안 함          │
  │ → 캐머런의 "자기 허락(Permission)" 원리                     │
  ├─────────────────────────────────────────────────────────────┤
  │ 계층 3: 위기 안전 (안전망으로 구현)                          │
  │ ─────────────────────────────────────────────               │
  │ · 위기 상담 안내 모든 페이지 하단 상시 노출                  │
  │ · 부정적 결과에 반드시 희망 메시지 + 전문기관 안내           │
  │ · 자연스러운 톤으로 전문 상담 연결                           │
  │ → 최민수 페르소나 기반 필수 설계                             │
  └─────────────────────────────────────────────────────────────┘

  ■ 각 화면 단계별 안전감 적용 포인트

  단계          안전감 메시지 위치              예시
  ═══════════════════════════════════════════════════════════════
  도입 화면     제목 바로 아래                  "지금 느끼는 감정,
                소요시간 위                     뭐든 괜찮아요."

  탐색 화면     카드 상단                       "정답은 없어요.
                                                편하게 살펴보세요."

  체험 화면     서술 입력란 위                  "(선택) 적고 싶은
                                                만큼만 적어보세요."
                입력란 아래                     "당신의 답은
                                                저장되지 않습니다."

  결과 화면     결과 카드 직후                  "이 워크숍을 해본 것
                                                자체가 자기를 돌보는
                                                첫 걸음이에요."
                실천 가이드 위                  "아래는 제안이에요.
                                                하고 싶은 것만
                                                해보세요."

  공유 화면     공유 버튼 근처                  "공유하지 않아도
                                                괜찮아요."

  위기 안내     모든 페이지 하단                "지금 힘든 순간이라면,
                                                혼자 감당하지 않아도
                                                됩니다."
  ═══════════════════════════════════════════════════════════════

6. 체험 워크플로우 — 풀 (v1 유지 + 아티스트 웨이 원리 삽입)
────────────────────────────────────────────────────────────────

  v1 4단계 구조 유지. 각 단계에 AW 원리 삽입점 표기:

  STEP 1 도입:
    + Permission 메시지 [v2]
    + "답은 저장되지 않습니다" [v2]

  STEP 2 탐색:
    + "정답은 없어요. 편하게 살펴보세요." [v2]
    → 캐머런: 탐색 자체가 목적 (결과가 아님)

  STEP 3 체험:
    + 서술형에 "(선택)" 명시 + "답은 저장되지 않습니다" [v2]
    + 서술 placeholder: "한 줄이라도, 한 단어라도 괜찮아요" [v2]
    → 캐머런: 모닝 페이지 = 양이 아니라 시작 자체가 핵심

  STEP 4 결과:
    + 검열자 대응 메시지 [v2]
    + 실천 가이드에 "제안이에요. 하고 싶은 것만 해보세요." [v2]
    + 실천 가이드에 아티스트 데이트 요소 포함 [v2]
      예: "좋아하는 카페에서 혼자 10분 앉아보기"
          "어린 시절 좋아했던 것 하나 떠올려보기"
