10. AI 클리셰 분석 및 제거 [v5 신설]

────────────────────────────────────────────────────────────────
10.1 AI가 자주 쓰는 시각 패턴 목록
────────────────────────────────────────────────────────────────

  ■ 이모지 남용 패턴

  AI가 과도하게 사용하는 이모지 (사용 빈도 순):
  ✨ (sparkles)     — AI 생성물의 90%에 등장
  🚀 (rocket)       — "시작하기" "런칭"에 반사적 사용
  💡 (lightbulb)    — "아이디어" "팁"에 기계적 사용
  ⚡ (lightning)     — "빠른" "에너지"에 반복 사용
  🔥 (fire)         — "핫" "인기"에 남용
  🎯 (target)       — "목표" "핵심"에 클리셰
  📝 (memo)         — "작성" "기록"에 무의식적 사용
  🎉 (party)        — "축하" "완료"에 과잉 사용

  AI의 전형적 패턴:
  "섹션 제목 앞에 이모지 1개 붙이기"를 거의 반사적으로 수행

  ■ SVG 아이콘 스타일

  AI 생성 SVG의 99%가 Lucide/Feather 계열:
  · stroke-width: 2, stroke-linecap: round, stroke-linejoin: round
  · fill: none, 24×24 viewBox
  · currentColor 사용
  → 이 조합 자체가 "AI가 만든 아이콘" 시그널

  ■ 레이아웃 클리셰

  패턴                       AI 생성 빈도    이 워크숍 해당 여부
  ═══════════════════════════════════════════════════════════════
  히어로 그라데이션 배경       95%             해당 ★
  물결 SVG 구분선             85%             해당 ★
  둥근 카드 (12~16px)         90%             해당 (유지 OK)
  fadeIn + translateY         95%             해당 (유지 OK)
  pulse scale 애니메이션      80%             해당 ★
  max-width 540px 중앙정렬   85%             해당 (유지 OK)
  ✧ · ✦ 별 장식 텍스트       70%             해당 ★
  완벽한 BEM + CSS 변수       90%             해당 (유지 OK)
  ═══════════════════════════════════════════════════════════════

────────────────────────────────────────────────────────────────
10.3 제거/교체 대상 및 대안
────────────────────────────────────────────────────────────────

  ■ AI-01: 섹션 제목 이모지 접두사 제거

  변경 전: "⚡ 1분 미니 워크숍" / "📝 5분 풀 워크숍"
  변경 후: "1분 미니 워크숍" / "5분 풀 워크숍"

  근거: 섹션 제목 앞 이모지는 AI의 가장 대표적 클리셰.
       제목 자체가 충분히 명확하므로 이모지 불필요.

  ■ AI-02: 히어로 그라데이션 → 단색

  변경 전: linear-gradient(180deg, #fff, var(--light))
  변경 후: background: #fff (단색 흰색)

  근거: 그라데이션 히어로는 AI 생성물의 95%에 등장.
       단색 흰색이 오히려 깨끗하고 차별화됨.
       캐릭터와 타이포그래피로 충분히 감성 전달.

  ■ AI-03: 물결 SVG 구분선 제거

  변경 전: <svg class="wave-divider">...</svg>
  변경 후: <div class="divider"></div> (점 장식 · · ·)

  근거: SVG 물결 구분선은 AI가 특히 좋아하는 장식 요소.
       서비스 전체에서 점 장식(· · ·)으로 통일.

  ■ AI-04: pulse 애니메이션 제거

  변경 전: .anim-pulse (scale 1→1.1 반복)
  변경 후: 이모지 정지 상태 (애니메이션 없음)

  근거: pulse는 AI 생성 UI의 80%에 등장하는 클리셰.
       정지 상태가 오히려 차분하고 심리 워크숍 톤에 맞음.
       bounce(캐릭터 등장)는 목적이 명확하므로 유지.

  ■ AI-05: "✧ · ✦" 별 장식 → 카테고리 장식으로 교체

  변경 전: "✧ · ✦" (결과 카드 상하단)
  변경 후: "— · —" (대시+점, 더 절제된 장식)

  근거: ✧✦ 별 장식은 AI가 카드·구분선에 반복 사용하는 패턴.
       대시+점은 인간 디자이너가 더 자주 사용하는 절제된 장식.

  ■ AI-06: 섹션 제목 인라인 SVG 아이콘 제거

  변경 전: <span class="icon"><svg>번개</svg></span> 1분 미니
  변경 후: 1분 미니 워크숍 (텍스트만)

  근거: 인라인 SVG + Lucide 스타일은 AI 시그널.
       제목은 텍스트만으로 충분히 의미 전달.
       풀 워크숍 목록의 좌측 아이콘은 유지 (기능적 구분 역할).


────────────────────────────────────────────────────────────────
10.4 유지해도 되는 것
────────────────────────────────────────────────────────────────

  요소                   AI 패턴?    유지 근거
  ═══════════════════════════════════════════════════════════════
  둥근 카드 (12~16px)    해당        보편적 디자인, AI 고유 아님
  fadeIn 전환             해당        기능적 필요 (단계 전환 인지)
  bounce 캐릭터 등장      해당        캐릭터 목적 명확, 1회만 실행
  CSS 변수 체계           해당        코드 유지보수에 실질적 유용
  max-width 540px        해당        모바일 우선의 표준 패턴
  BEM 네이밍             해당        구조적 명확성
  선택지 내 이모지       아님        의미 전달 역할 (날씨=감정)
  결과 카드 내 이모지    아님        결과 표현 역할 (☁=흐림)
  풀 목록 좌측 SVG       일부        기능적 구분 (카테고리 시각화)
  ═══════════════════════════════════════════════════════════════

  ★ "AI가 쓰는 패턴"이라고 다 없앨 필요는 없음.
    보편적·기능적인 것은 유지, "AI만의 클리셰"만 제거.

11.1 필요성 분석 (5개 관점)
────────────────────────────────────────────────────────────────

  관점              분석                           결론
  ═══════════════════════════════════════════════════════════════
  심리 워크숍       아티스트 웨이 원칙:             불필요
  철학              "검열 없이 앞으로 나아가기"
                    이전 버튼 = 자기검열/반추 유도

  답 변경 니즈      체험 단계 내에서 radio 재클릭   불필요
                    가능. 결과→체험은 뒤로가기.     (내장 대체)

  흐름 방해         이전/다음 2버튼 → 선택 증가     불필요
                    → 의사결정 지연                 (깔끔한 1버튼)
                    → 완주율 15~25% 감소

  플로우 길이       4단계뿐 (도입→탐색→체험→결과)   불필요
                    20문항 테스트가 아님             (짧아서 충분)

  뒤로가기 대체     :target 앵커가 브라우저 히스토리  충분
                    에 쌓임. Back 버튼으로 복귀 가능.  (은닉 경로)
  ═══════════════════════════════════════════════════════════════


────────────────────────────────────────────────────────────────
11.2 최종 결정: 이전 버튼 미추가
────────────────────────────────────────────────────────────────

  ★ 이전 버튼을 추가하지 않는다.

  근거:
  [1] "감정을 알아차리는 체험"이므로 되돌아가서 최적화할 대상이 없음
  [2] 이전 버튼은 "아까 고른 거 맞나?" 자기검열을 유도 → 치료적 흐름 훼손
  [3] 브라우저 뒤로가기가 은닉 경로로서 충분히 기능함
  [4] 버튼 1개(다음만)의 깔끔한 인터페이스가 인지 부하 최소
  [5] 4단계 짧은 플로우에서 이전 버튼은 과잉 설계


────────────────────────────────────────────────────────────────
11.3 ADR-013 이전 버튼 미추가
────────────────────────────────────────────────────────────────

  ADR-013: 워크숍 진행 시 "이전" 버튼을 추가하지 않는다.
  ────────────────────────────────────────────
  결정: 단방향 흐름(다음만) 유지, 이전 버튼 미추가
  근거: 아티스트 웨이 "앞으로 나아가기" 원칙, 완주율 보호,
       브라우저 뒤로가기로 대체 가능
  트레이드오프: 일부 사용자가 이전 방법을 모를 수 있음
              → 향후 필요 시 헤더에 작은 "< 뒤로" 텍스트 링크 검토
              → 현 시점에서는 미추가
