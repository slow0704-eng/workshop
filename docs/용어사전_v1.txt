■ 감정 계층 모델 [심리]
  사용자의 표면 행동 → 표면 감정 → 중간 니즈 → 깊은 니즈를
  4단계로 분석하는 프레임워크.
  예: "심리테스트 해볼까"(표면) → "나는 괜찮은 사람인가"(깊은)
  출처: 유저페르소나 v2.0

■ 검열자 → "내면의 검열자" 참조

■ 검열자 대응 메시지 [심리][콘텐츠]
  결과 화면에서 사용자의 내면 검열자("해봤자 뭐")에 대해
  부드럽게 반박하는 메시지.
  예: "여기까지 온 것만으로도 나를 돌보고 있는 거야."
  위치: 결과 카드 직후, 캐릭터 말풍선으로 전달.
  출처: 워크플로우모델 v2.0, 벤치마킹분석서

■ 결과 카드 [콘텐츠][기술]
  워크숍 완료 후 표시되는 정사각형(1:1) 비주얼 카드.
  사용자의 선택 결과를 감성적 문구 + 캐릭터로 표현.
  스크린샷/이미지 저장하여 SNS 공유하는 것이 핵심 목적.
  CSS class: .result-card
  사양: aspect-ratio 1/1, max-width 360px, border-radius 16px
  출처: 와이어프레임 v1.0 12번, v2.0 16번

■ 결과 카드 워터마크 [콘텐츠]
  결과 카드 하단에 작게 표시되는 서비스 URL.
  SNS 공유 시 자연스러운 서비스 홍보 역할.
  예: "maum-workshop.github.io"
  사양: 12px, #BBB

■ 골라 쓰기 [콘텐츠][기술]
  서술형(유형 C) 워크숍에서 직접 글을 쓰는 대신
  미리 제공된 선택지 중 하나를 고르는 방식.
  서술 피로를 해소하며, 선택지 자체가 "롤모델 메시지" 역할.
  CSS 구현: radio + :checked 탭 전환
  출처: 워크플로우모델 v3.0, 콘텐츠매트릭스 v2.0

■ 공동 브랜딩 [유통]
  축제·기관과 협업 시 기관 로고 + 마음 워크숍 로고를 함께 표시하는 것.
  /event/ 폴더의 행사 전용 HTML 파일로 구현.
  출처: 사업계획서 v3.0, 워크플로우모델 v1.0

■ 구름 마음이 → "마음이" 참조

■ 나를 만나는 12일 [콘텐츠]
  아티스트 웨이 12주 프로그램을 12일로 축소한 시그니처 챌린지(C04).
  매일 5분, 12개 주제를 순차 체험. 총 60분 = 원작 42시간의 1/42.
  체험 유형 분포: A×4 + B×5 + C×3.
  출처: 벤치마킹분석서, 콘텐츠매트릭스 v2.0

■ 내면의 검열자 (Inner Critic / Censor) [심리]
  아티스트 웨이 핵심 개념. 자기 안에서 모든 시도를 비판하는 목소리.
  "넌 안 돼", "해봤자 뭐가 달라지겠어" 같은 내면 대화.
  마음 워크숍에서는 이 목소리를 인식 → 분리 → 대응하는 체험 제공.
  관련 워크숍: F21 비판자 알아차리기, F05 비판자에게 편지, M12
  출처: 벤치마킹분석서, 유저페르소나 v2.0

■ 리커트 척도 (Likert Scale) [콘텐츠][기술]

  심리검사 스타일의 전문적 느낌을 부여.
  CSS 구현: radio×5 가로 배치, 원형 버튼, 44px 터치 영역.
  사용 워크숍: M03 자존감 온도, M04 스트레스 레벨, M07 회복력, F19 성장 마인드셋
  출처: 워크플로우모델 v3.0, 와이어프레임 v2.0 8번

■ 롤모델 메시지 [심리][콘텐츠]
  "골라 쓰기" 선택지로 제공되는 사전 제작 문구.
  사용자에게 "이렇게 생각해도 되는구나"의 모범 예시 역할.
  예: "고마워, 하지만 이제는 내가 결정할게." (비판자 대응)
  예: "지금 고생하고 있지만, 다 잘 될 거야." (5년 후 엽서)
  출처: 콘텐츠매트릭스 v2.0 8번

■ 마음 워크숍 [서비스]
  본 서비스의 명칭 (가칭).
  5분 이내 완료 가능한 심리·자기탐색 체험형 웹 워크숍.
  무료 공공 서비스, HTML+CSS only, QR 즉시 접속.
  슬로건: "5분, 나를 만나는 시간"

■ 마음이 [캐릭터]
  서비스 전체에 등장하는 안내 캐릭터.
  형태: 둥글고 부드러운 구름 모양.
  색상: 카테고리별 연한 색상으로 변화.
  크기: 48px(기본), 36px(결과카드 내), 64px(히어로), 20px(헤더).
  역할: 안내자(워크숍 진행) / 공감자(선택 반응) / 응원자(결과 격려).
  표정: 7종 (기본미소, 반가움, 경청, 공감, 응원, 위로, 축하).
  구현: SVG 파일 또는 CSS, /img/maumi-표정명.svg
  톤: 반말 + 따뜻함 + 판단 없음.
  예외: 위기 상담 안내 영역에는 등장하지 않음.
  출처: 워크플로우모델 v3.0, 와이어프레임 v2.0 3~4번

■ 마이크로 페이지 [콘텐츠][심리]
  아티스트 웨이의 "모닝 페이지"(매일 3페이지 자유 글쓰기)를
  5분 이내 축소한 마음 워크숍 버전.
  <textarea>에 자유 서술하는 체험. 항상 "(선택)" 표시.
  새로고침 시 삭제 = "아무도 읽지 않는" 안전감.
  출처: 벤치마킹분석서, 워크플로우모델 v2.0

■ 말풍선 [캐릭터][기술]
  마음이 캐릭터의 대사를 표시하는 UI 컴포넌트.
  배경: #F8F9FA, 둥근 모서리 12px, padding 14px 16px.

  대사: 14px, #555, line-height 1.6.

■ 바이럴 루프 [유통]

  온라인 루프: SNS 결과 카드 공유 → 링크 클릭 → 새 사용자.
  오프라인-온라인 루프: 축제 QR → 현장 체험 → SNS 공유 → 온라인 유입.

■ 반응 대사 [캐릭터][콘텐츠]
  사용자가 선택지를 탭했을 때 캐릭터가 즉시 표시하는 한 줄 대사.
  긍정 선택: 응원 ("좋은 아침이었나 보다!")
  중립 선택: 인정 ("그렇구나.")
  부정 선택: 공감 ("좀 힘든 하루였나 봐. 괜찮아.")
  CSS 구현: :checked + 형제 선택자로 표시/숨김 전환.
  출처: 워크플로우모델 v3.0, 와이어프레임 v2.0 4번

■ 사용 환경 (Context of Use) [서비스]
  사용자가 워크숍을 이용하는 물리적·심리적 맥락.
  A: 청년축제·행사 현장 / B: 개인 여유 시간 / C: 직장 점심시간
  D: 학교 상담실 / E: 혼자 있는 밤 시간 (v2 추가)
  출처: 유저페르소나 v1.0~v2.0

■ 선택형 (유형 A) [콘텐츠]
  모든 체험이 선택지(radio/checkbox/리커트/카드)로만 구성된 워크숍.
  서술 입력 없음 → 서술 피로 제로.
  전체 콘텐츠의 50% 차지.
  출처: 워크플로우모델 v3.0, 콘텐츠매트릭스 v2.0

■ 서술대안형 (유형 C) [콘텐츠]
  서술이 핵심이지만 "직접 쓰기" 또는 "골라 쓰기" 중 택 1 가능한 워크숍.
  전체 콘텐츠의 19% 차지. 실질적으로 선택만으로도 체험 완료 가능.
  CSS 구현: radio 탭 전환으로 textarea ↔ 선택지 영역 전환.
  출처: 워크플로우모델 v3.0

■ 셀프 데이트 → "아티스트 데이트" 참조

■ 심리적 안전감 레이어 [심리][서비스]
  모든 화면에 관통 적용되는 보이지 않는 설계 구조.
  3계층: 기술적 안전(데이터 미수집) + 정서적 안전(Permission 메시지)
  + 위기 안전(상담 안내 상시 노출).
  출처: 워크플로우모델 v2.0

■ 실천 가이드 [콘텐츠]
  풀 워크숍 결과 화면에서 제공하는 구체적 행동 제안 3가지.
  원칙: "지금 바로 할 수 있는 정말 작은 것"
  아티스트 데이트 요소 포함 (혼자만의 시간, 어린 시절 좋아한 것 등).
  톤: "이건 그냥 제안이야. 하고 싶은 것만 해봐." (강제 없음)
  출처: 사업계획서 v3.0, 벤치마킹분석서

■ 아티스트 데이트 (Artist Date) [심리]
  아티스트 웨이의 2대 도구 중 하나.
  매주 1회 혼자만의 2시간을 보내는 자기 돌봄 시간.
  마음 워크숍에서는 "5분 셀프 데이트"로 축소 재해석.
  슬로건 "5분, 나를 만나는 시간" 자체가 아티스트 데이트의 축소판.
  실천 가이드에 "좋아하는 카페에서 혼자 10분" 등으로 반영.
  출처: 벤치마킹분석서

■ 아티스트 웨이 (The Artist's Way) [심리]
  줄리아 캐머런(Julia Cameron) 저, 1992년 초판.
  12주 창의성 회복 프로그램. 마음 워크숍의 핵심 영감 출처.
  2대 도구: 모닝 페이지 + 아티스트 데이트.
  마음 워크숍은 이 원리를 한국 청년 맞춤 5분 웹 워크숍으로 재구성.
  ★ 원작 텍스트 직접 인용은 저작권상 불가.
  출처: 벤치마킹분석서

■ 안전한 공간 (Safe Container) [심리]
  아티스트 웨이 핵심 개념. 판단 없이 자기를 드러낼 수 있는 환경.
  마음 워크숍 구현: 데이터 미수집 + 광고 없음 + 회원가입 없음
  + "당신의 답은 저장되지 않습니다" 안내.
  캐머런의 "아무도 읽지 않는 모닝 페이지" = 마음 워크숍의 안전한 공간.
  출처: 벤치마킹분석서, 유저페르소나 v2.0

  HTTPS 기본 지원, CDN 서빙, git push만으로 자동 배포.
  마음 워크숍의 기본 호스팅 플랫폼.
  출처: 사업계획서 v2.0~v3.0

■ HTML-only [기술]
  JavaScript를 사용하지 않고 순수 HTML + CSS만으로 구현하는 방식.
  마음 워크숍의 핵심 기술 원칙.
  장점: 초경량, 보안(XSS 불가), 크로스브라우저, 유지보수 제로.
  인터랙션: :target(단계 전환) + :checked(결과 분기) + <details>(접기)
  Phase 3에서 일부 JS(html2canvas, Clipboard API) 선택적 도입 검토.
  출처: 사업계획서 v2.0
