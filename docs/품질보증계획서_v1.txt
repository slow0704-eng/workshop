5. 결과 분기 테스트
────────────────────────────────────────────────────────────────

  ★ 가장 중요한 테스트 — 모든 선택 조합에서 올바른 결과 확인

  ■ 테스트 매트릭스 (워크숍당)

  워크숍 유형       선택지 수    결과 분기 수    테스트 케이스 수
  ═══════════════════════════════════════════════════════════════
  미니 (radio×4Q)   4질문×3~4    3~4개 결과     3~4건
  풀 B (radio)      1핵심 질문    4~5개 결과     4~5건
  풀 C (탭+radio)   1핵심+탭     4~5개 결과     8~10건
  리커트 (대표1)    1대표 질문    3~5개 결과     3~5건
  ═══════════════════════════════════════════════════════════════

  ■ 테스트 절차 (워크숍 1개당)

  1. 선택지 A 선택 → "결과 보기" → 결과 A 메시지 정상 표시 확인
  2. 새로고침 → 선택지 B 선택 → 결과 B 메시지 확인
  3. 모든 선택지에 대해 반복
  4. 결과 카드가 정사각형으로 표시되는지 확인
  5. 캐릭터 표정이 결과에 맞게 표시되는지 확인
  6. 부정적 결과에 희망 메시지가 포함되었는지 확인

  ■ MVP 5개 워크숍 테스트 케이스 수

  M01 에너지 잔량:      결과 4개 × 1 = 4건
  M02 감정 체크:        결과 3개 × 1 = 3건
  F01 감정 날씨:        결과 5개 × 1 = 5건
  F02 강점 발견:        결과 6개 × 1 = 6건
  F03 5년 후 엽서:      결과 5개 + 탭 2 = 7건
  ─────────────────────────────────────────
  합계:                              25건

      2. Throttling: "Slow 3G" 선택
      3. "Disable cache" 체크
      4. 페이지 새로고침
      5. 확인 항목:
         □ 전체 로딩 시간 1초 이내
         □ HTML 파일 크기 30KB 이하
         □ CSS 합계 25KB 이하
         □ SVG 개당 10KB 이하
         □ 전체 페이지 200KB 이하
         □ 외부 도메인 요청 0건

  [2] Lighthouse 테스트
      1. DevTools → Lighthouse 탭
      2. Categories: Performance + Accessibility + SEO 체크
      3. Device: Mobile 선택
      4. "Analyze page load" 실행
      5. 확인 항목:
         □ Performance 점수 95+
         □ Accessibility 점수 90+
         □ SEO 점수 90+
         □ First Contentful Paint 1초 이내

  [3] 애니메이션 성능 테스트
      1. DevTools → Performance 탭
      2. 워크숍 체험 전 과정 녹화
      3. 확인 항목:
         □ 프레임 드롭 없음 (60fps 유지)
         □ fadeIn/bounce 애니메이션 부드러움

────────────────────────────────────────────────────────────────
17. 콘텐츠 검수 체크리스트 (워크숍 1개당)
────────────────────────────────────────────────────────────────

  ■ 톤앤매너 검수

  □ 캐릭터 대사가 반말 + 따뜻한 톤인가?
  □ "~해도 괜찮아" 류의 Permission 메시지가 도입에 있는가?
  □ 서술 영역에 "(선택)" 표시가 있는가?
  □ "답은 저장되지 않습니다" 안내가 있는가?
  □ 실천 가이드가 "제안" 톤인가? ("이건 그냥 제안이야")
  □ "공유 안 해도 괜찮아" 톤이 있는가?
  □ 위기 안내는 존댓말 + 진지한 톤인가?

  ■ 아티스트 웨이 원리 검수

  □ 도입에 자기 허락(Permission) 요소가 있는가?
  □ 결과에 검열자 대응 메시지가 있는가?
  □ 부정적 결과에 "당신 잘못이 아니에요" 류가 있는가?
  □ 실천 가이드에 아티스트 데이트 요소가 있는가?
     (혼자만의 시간, 어린 시절 좋아한 것 등)
  □ AW 태그가 부여되었는가?

  ■ 캐릭터 검수

  □ 도입 대사가 자연스러운가? (반말, 친근)
  □ 선택별 반응 대사가 적절한가? (긍정→응원, 부정→공감)
  □ 결과 대사가 공감적인가?
  □ 위기 안내 영역에 캐릭터가 없는가?
  □ 결과 카드 내에 캐릭터(36px)가 포함되어 있는가?

  ■ 콘텐츠 구조 검수

  □ 모든 선택 조합에서 결과가 정상 표시되는가?
  □ 실천 가이드 3가지가 "정말 작은 행동"인가?
  □ 추천 워크숍 링크 2개가 올바른 페이지로 연결되는가?
  □ 추천에 체험 유형 뱃지가 표시되는가?
  □ 유형 C인 경우 골라쓰기 선택지 4~5개가 있는가?
  □ 골라쓰기 선택지가 롤모델 메시지로 적절한가?


────────────────────────────────────────────────────────────────
18. 결과 카드 디자인 검수
────────────────────────────────────────────────────────────────

  □ 정사각형 비율 (1:1)
  □ max-width 360px
  □ 카테고리별 색상 적용
  □ 캐릭터(36px) 카드 내 포함
  □ URL 워터마크 하단 표시
  □ 스크린샷 캡처 시 카드 전체가 한 화면에 보임
  □ 인스타그램 스토리 업로드 시 잘리지 않음
  □ 결과 문구가 "공유하고 싶은 매력"이 있는가?
     ("당신의 에너지 잔량: 23% ☁" 수준)

19. OG 메타 태그 검수

  □ og:title 설정됨
  □ og:description 설정됨
  □ og:image 설정됨 (캐릭터 포함 이미지)
  □ og:url 설정됨

  □ 미리보기에 제목 + 설명 + 캐릭터 이미지 확인

20. 사용성 테스트 계획
────────────────────────────────────────────────────────────────

  ■ 시점: Phase 1 MVP 완료 후 (배포 전 또는 직후)

  ■ 참여자: 지인 3~5명
  - 20대 대학생 1~2명 (김하늘 대응)
  - 20대 후반~30대 직장인 1~2명 (이준호 대응)
  - 심리·자기계발에 관심 있는 사람 1명

  ■ 테스트 시나리오

  시나리오 1: 첫 방문 체험 (미니)
  ─────────────────────────────────────────
  1. "이 QR을 스캔해보세요" (또는 URL 전달)
  2. 미니 워크숍 하나를 해보세요
  3. 관찰: 어디서 멈추는가? 혼란스러운 곳은?
  4. 질문: "이건 뭐라고 생각했어요?", "어려운 곳 있었어요?"

  시나리오 2: 결과 공유
  ─────────────────────────────────────────
  1. 결과 카드를 본 소감
  2. "공유하고 싶은 느낌이 드나요?"
  3. 실제 스크린샷/링크 공유 시도
  4. 관찰: 공유 방법을 바로 찾는가?

  시나리오 3: 풀 워크숍 전환
  ─────────────────────────────────────────
  1. 미니 결과에서 풀 워크숍 전환 링크 발견하는지
  2. 풀 워크숍 체험 (서술 영역 반응 관찰)
  3. "서술을 적고 싶었나요? 부담스러웠나요?"

  ■ 수집 항목

  - 완료율: 시작 → 결과까지 도달했는가?
  - 이탈 지점: 어디서 멈추거나 혼란스러워했는가?
  - 소요 시간: 실제 체험에 걸린 시간
  - 결과 공유 의향: 캡처/공유를 시도했는가?
  - 감성 반응: 캐릭터 대사, 결과 메시지에 대한 감상
  - 개선 의견: 자유 피드백
