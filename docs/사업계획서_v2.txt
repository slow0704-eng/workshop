1. 사업 개요
────────────────────────────────────────────────────────────────

■ 서비스명
  마음 워크숍 (가칭)

■ 서비스 형태
  워크숍 북 형태의 체험형 웹사이트 (모바일 반응형)

■ 핵심 콘셉트
  5분 이내로 완료할 수 있는 자기계발·심리 워크숍을
  디지털 워크북 형태로 제공하는 무료 공공 서비스

■ 슬로건
  "5분, 나를 만나는 시간"

■ 운영 모델
  공공 목적 무료 서비스 (사회적 가치 창출 중심)

■ 개발 원칙 [v2 신설]
  - JavaScript 없이 순수 HTML + CSS만으로 구현
  - 코딩 지식 최소화 → 콘텐츠 작성에 집중
  - 빌드·번들링·프레임워크 일절 불필요
  - 파일 하나 = 워크숍 하나 (완전 독립 구조)

2. 문제 정의 및 필요성
────────────────────────────────────────────────────────────────

■ 현황 및 문제
  - 직장인 83.6%가 번아웃을 경험 (2025 잡코리아 조사)
  - 대학생 우울감 경험 비율 지속 증가 추세
  - 심리상담 비용 부담 및 접근성 문제
  - 기존 자기계발 콘텐츠는 긴 호흡 → 바쁜 현대인에게 부적합

■ 해결 방향
  - 5분 이내 마이크로 워크숍으로 심리적 접근성 확보
  - 무료 제공으로 경제적 장벽 제거
  - 워크북 형태의 능동적 참여로 실질적 변화 유도
  - 언제 어디서든 접속 가능한 웹 기반 서비스

4. 서비스 구성 [v2 수정]
────────────────────────────────────────────────────────────────

■ 워크숍 구조 (1회 5분 이내)

  ★ HTML-only 구현: CSS의 :target, :checked, <details> 등을
    활용하여 JavaScript 없이 단계별 전환 구현

  [STEP 1] 도입 (30초)
      - 주제 소개 및 오늘의 질문 제시
      - CSS 애니메이션을 활용한 감성적 비주얼
      - 구현: 앵커 링크(#step1)로 페이지 내 이동

  [STEP 2] 탐색 (1분 30초)
      - 짧은 심리 개념 설명
      - 카드 형태 레이아웃 (CSS grid/flexbox)
      - 구현: <details><summary> 태그로 접기/펼치기
        또는 :target 선택자로 카드 전환

  [STEP 3] 체험 (2분)
      - 선택형 질문: radio/checkbox 입력 활용
      - 자기 진단: CSS :checked 선택자로 선택 시 시각 피드백
      - 자유 서술: <textarea> 입력란 제공 (저장 없이 체험용)
      - 구현: <form> + <input type="radio"> + CSS로
        선택에 따른 결과 표시 (JS 불필요)

  [STEP 4] 정리 (1분)
      - 선택 결과에 따른 메시지 표시 (CSS :checked + 형제 선택자)
      - 핵심 메시지 요약 카드
      - SNS 공유 링크 제공 (URL 기반)

■ HTML-only 인터랙션 기법 상세 [v2 신설]

  기법                    용도                    예시
  ─────────────────────────────────────────────────────────────
  :target 선택자          단계별 화면 전환        #step1, #step2 앵커
  :checked + ~ 선택자     선택에 따른 결과 표시   라디오 버튼 심리 테스트
  <details><summary>      정보 접기/펼치기        심리 개념 카드 열기
  <input type="range">    슬라이더 입력           감정 강도 체크
  CSS 애니메이션          시각 효과               페이드인, 슬라이드
  <progress>              진행률 표시             워크숍 진행 단계 바
  CSS scroll-snap         카드 넘기기 효과        좌우 스와이프 탐색

■ 콘텐츠 카테고리 (예시)

  카테고리            예시 워크숍
  ─────────────────────────────────────────────
  감정 관리           "오늘의 감정 날씨 체크"
                      "화가 날 때 3단계 호흡법"
  자존감              "나의 강점 5가지 발견하기"
                      "내면의 비판자에게 편지쓰기"
  관계                "경청 실험: 3분 대화법"
                      "감사 일기 한 줄 쓰기"
  스트레스 관리       "지금 이 순간 마인드풀니스"
                      "업무 우선순위 매트릭스"
  진로·목표           "5년 후 나에게 보내는 엽서"
                      "가치관 카드 정렬하기"
  회복탄력성          "실패 리프레이밍 워크숍"
                      "작은 성취 기록장"

5. 핵심 기능 [v2 대폭 수정]
────────────────────────────────────────────────────────────────

  [1] CSS-only 인터랙티브 워크북
      - :target, :checked, <details> 등 순수 CSS/HTML 기법
      - 라디오 버튼 + CSS로 선택형 심리 테스트 구현
      - CSS scroll-snap으로 카드 넘기기 체험
      - 모바일 터치 대응 (CSS 기본 지원)
      - ★ JavaScript 완전 배제 → 브라우저 호환성 극대화

  [2] SNS 공유 (URL 기반) [v2 간소화]
      - 워크숍별 고유 URL로 공유 (별도 기능 불필요)
      - Open Graph 메타 태그로 SNS 미리보기 자동 생성
      - 공유 링크: 카카오톡·트위터(X)·페이스북 URL scheme 활용
        (예: https://twitter.com/intent/tweet?url=... )
      - "나도 해보기" 링크 = 워크숍 페이지 URL 그 자체
      - ★ 결과 카드 이미지 생성 기능은 제거
        → 대신 결과 화면 자체를 스크린샷 유도 (안내 문구)
        → 향후 JS 도입 시 html2canvas 추가 가능

  [3] 개인 기록 [v2 간소화]
      - v2에서는 서버·DB·localStorage 모두 미사용
      - 자유 서술 입력은 체험용 (새로고침 시 초기화)
      - 기록 보관이 필요한 사용자에게는 안내:
        "스크린샷을 찍어 보관하세요"
        "결과를 메모 앱에 복사해 두세요"
      - ★ 향후 확장 시 localStorage 또는 외부 연동 검토

  [4] 워크숍 탐색 [v2 간소화]
      - 메인 페이지에서 카테고리별 HTML 앵커 링크로 분류
      - CSS :target으로 카테고리 필터링 효과 구현
      - "오늘의 추천"은 주기적으로 메인 HTML 수동 업데이트
      - ★ 자동 추천·개인화 기능은 제외 (JS 필요)

  [5] 초경량 운영 설계 [v2 강화]
      - 서버·DB·JS 없이 순수 HTML+CSS만으로 운영
      - 콘텐츠 추가 = HTML 파일 1개 복사 + 내용 교체
      - 유지보수 업무 완전 제로 수준
      - 장애 가능성 사실상 없음 (정적 파일만 서빙)

8. 기술 스택 및 개발 계획 [v2 전면 개편]
────────────────────────────────────────────────────────────────

■ 개발 체제
  - 1인 개발 (기획·디자인·개발·운영 겸임)
  - HTML만 작성할 줄 알면 워크숍 추가 가능한 구조

■ 기술 구성

  구분           기술
  ─────────────────────────────────────────────
  핵심 구조      순수 HTML + CSS
                 ★ JavaScript 완전 배제
                 ★ 프레임워크·빌드 도구 일절 불필요
  인터랙션       CSS :target, :checked, <details>
                 CSS scroll-snap, 애니메이션
                 HTML <form>, <input>, <textarea>
  결과 표시      CSS :checked + 형제/자식 선택자로
                 선택에 따른 결과 메시지 표시
  호스팅         GitHub Pages (무료, 무서버)
                 또는 Netlify / Vercel 정적 호스팅
  SNS 공유       Open Graph 메타 태그
                 URL scheme 기반 공유 링크
                 (카카오 JS SDK 미사용 → URL 직접 공유)
  데이터 저장    없음 (서버·DB·localStorage 모두 미사용)
                 ★ 사용자 안내: 스크린샷 또는 메모로 보관

  ★ 기술 선택 근거 (v2 강화)
  - JS 제거 → 개발 시간 50% 이상 단축
  - JS 제거 → 디버깅·크로스브라우저 이슈 제거
  - JS 제거 → 보안 취약점 원천 차단 (XSS 불가)
  - 서버 없음 → 서버 비용 0원, 장애 가능성 제로
  - 빌드 없음 → HTML 파일 수정만으로 콘텐츠 추가
  - 1인 운영 → 배울 것 최소, HTML만 알면 운영 가능
  - 정적 사이트 → 트래픽 급증에도 안정적 (CDN)
  - 접근성 향상 → JS 비활성화 환경에서도 완전 작동

■ 워크숍 HTML 템플릿 구조 예시 [v2 신설]

  하나의 워크숍 HTML 파일 내부 구조:

  ┌─────────────────────────────────────────┐
  │ <head>                                  │
  │   OG 메타 태그 (SNS 공유용)             │
  │   공통 CSS 링크                         │
  │ </head>                                 │
  │                                         │
  │ <section id="step1"> 도입 </section>    │
  │   → "다음" 앵커 링크 → #step2           │
  │                                         │
  │ <section id="step2"> 탐색 </section>    │
  │   → <details> 태그로 카드 펼치기        │
  │   → "다음" 앵커 링크 → #step3           │
  │                                         │
  │ <section id="step3"> 체험 </section>    │
  │   → <form> + radio/checkbox 입력        │
  │   → CSS :checked로 실시간 피드백        │
  │   → <textarea> 자유 서술                │
  │   → "결과 보기" 앵커 링크 → #result     │
  │                                         │
  │ <section id="result"> 결과 </section>   │
  │   → CSS :checked 상태에 따른 결과 표시  │
  │   → 핵심 메시지 요약                    │
  │   → SNS 공유 URL 링크                   │
  │   → 다른 워크숍 추천 링크               │
  └─────────────────────────────────────────┘

■ 개발 로드맵 (1인 기준) [v2 대폭 단축]

  Phase 1 (3~5일) — MVP 출시 ★ 기존 2~3주 → 1주 이내
  ├── 공통 CSS 스타일시트 제작 (1일)
  │   ├── 반응형 레이아웃 (모바일 우선)
  │   ├── :target 기반 단계 전환 스타일
  │   ├── :checked 기반 결과 표시 스타일
  │   └── 애니메이션 (페이드, 슬라이드)
  ├── 워크숍 HTML 템플릿 제작 (1일)
  │   └── 복사해서 내용만 바꾸면 되는 표준 템플릿
  ├── 핵심 워크숍 3개 제작 (1~2일)
  │   ├── "오늘의 감정 날씨 체크" (선택형)
  │   ├── "나의 강점 발견하기" (체크리스트형)
  │   └── "5년 후 나에게 보내는 엽서" (서술형)
  ├── 메인 페이지 (index.html) 제작 (반나절)
  ├── OG 메타 태그 + SNS 공유 링크 설정 (반나절)
  └── GitHub Pages 배포

  Phase 2 (2~3주) — 콘텐츠 확장 ★ 기존 1~2개월 → 3주 이내
  ├── 워크숍 10개로 확장
  │   (템플릿 복사 방식 → 1개당 2~3시간)
  ├── 메인 페이지 카테고리별 앵커 분류
  ├── OG 이미지 카테고리별 제작 (Canva 활용)
  └── SEO 기본 최적화 (메타 태그, 시맨틱 HTML)

  Phase 3 (1~2개월) — 안정화 및 확산 ★ 기존 3~4개월 → 2개월 이내
  ├── 워크숍 20개 이상 확보
  ├── 시리즈형 워크숍 구성 (링크로 순서 연결)
  ├── 사용자 피드백 수집 (구글 폼 외부 링크)
  ├── 기관 제휴 및 홍보 활동
  └── (선택) JS 부분 도입 검토
      ├── 결과 카드 이미지 생성 (html2canvas)
      ├── localStorage 기반 기록 저장
      └── 카카오톡 공유 API 연동

  ★ 총 개발 기간 비교
  ─────────────────────────────────────────────
  구분          v1 (HTML+CSS+JS)    v2 (HTML+CSS only)
  ─────────────────────────────────────────────

  안정화        3~4개월             1~2개월

  ─────────────────────────────────────────────

  5. git push → 자동 배포 완료
  → 전체 소요시간: 2~3시간 (콘텐츠 기획 포함)
