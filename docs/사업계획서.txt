================================================================
        워크숍 북 체험형 웹사이트 사업계획서
        "마음 워크숍" (가칭)
================================================================

작성일: 2026년 3월 14일

────────────────────────────────────────────────────────────────
1. 사업 개요
────────────────────────────────────────────────────────────────

■ 서비스명
  마음 워크숍 (가칭)

■ 서비스 형태
  워크숍 북 형태의 체험형 웹사이트 (모바일 반응형)

■ 핵심 콘셉트
  5분 이내로 완료할 수 있는 자기계발·심리 워크숍을
  디지털 워크북 형태로 제공하는 무료 공공 서비스

■ 슬로건
  "5분, 나를 만나는 시간"

■ 운영 모델
  공공 목적 무료 서비스 (사회적 가치 창출 중심)


────────────────────────────────────────────────────────────────
2. 문제 정의 및 필요성
────────────────────────────────────────────────────────────────

■ 현황 및 문제
  - 직장인 83.6%가 번아웃을 경험 (2025 잡코리아 조사)
  - 대학생 우울감 경험 비율 지속 증가 추세
  - 심리상담 비용 부담 및 접근성 문제
  - 기존 자기계발 콘텐츠는 긴 호흡 → 바쁜 현대인에게 부적합

■ 해결 방향
  - 5분 이내 마이크로 워크숍으로 심리적 접근성 확보
  - 무료 제공으로 경제적 장벽 제거
  - 워크북 형태의 능동적 참여로 실질적 변화 유도
  - 언제 어디서든 접속 가능한 웹 기반 서비스


────────────────────────────────────────────────────────────────
3. 대상 고객 (타겟 사용자)
────────────────────────────────────────────────────────────────

■ 주요 타겟

  [1] 직장인 (25~45세)
      - 업무 스트레스 및 번아웃 관리 필요
      - 점심시간·출퇴근 시간 활용 가능한 짧은 콘텐츠 선호
      - 자기계발에 관심 있으나 시간 부족

  [2] 대학생·취준생 (18~28세)
      - 진로 불안, 자존감 저하 등 심리적 어려움
      - 디지털 네이티브로 웹 서비스 친화적
      - SNS 공유에 적극적인 세대

■ 보조 타겟
  - 심리·자기계발에 관심 있는 일반 성인
  - 기업 HR 담당자 (사내 복지 프로그램 연계)
  - 학교 상담실·청소년센터 등 공공기관


────────────────────────────────────────────────────────────────
4. 서비스 구성
────────────────────────────────────────────────────────────────

■ 워크숍 구조 (1회 5분 이내)

  [STEP 1] 도입 (30초)
      - 주제 소개 및 오늘의 질문 제시
      - 감성적 일러스트 또는 짧은 영상

  [STEP 2] 탐색 (1분 30초)
      - 짧은 심리 개념 설명 (카드 넘기기 형태)
      - 관련 사례 또는 비유 제시

  [STEP 3] 체험 (2분)
      - 인터랙티브 워크시트 작성
      - 선택형 질문, 슬라이더, 자유 서술 등
      - 간단한 심리 테스트 또는 자기 진단

  [STEP 4] 정리 (1분)
      - 나만의 결과 카드 생성
      - 핵심 메시지 요약
      - SNS 공유 유도

■ 콘텐츠 카테고리 (예시)

  카테고리            예시 워크숍
  ─────────────────────────────────────────────
  감정 관리           "오늘의 감정 날씨 체크"
                      "화가 날 때 3단계 호흡법"
  자존감              "나의 강점 5가지 발견하기"
                      "내면의 비판자에게 편지쓰기"
  관계                "경청 실험: 3분 대화법"
                      "감사 일기 한 줄 쓰기"
  스트레스 관리       "지금 이 순간 마인드풀니스"
                      "업무 우선순위 매트릭스"
  진로·목표           "5년 후 나에게 보내는 엽서"
                      "가치관 카드 정렬하기"
  회복탄력성          "실패 리프레이밍 워크숍"
                      "작은 성취 기록장"


────────────────────────────────────────────────────────────────
5. 핵심 기능
────────────────────────────────────────────────────────────────

  [1] 인터랙티브 워크북 엔진
      - 카드 넘기기, 드래그앤드롭, 슬라이더 등 다양한 UI
      - 사용자 입력에 따른 맞춤형 결과 생성
      - 모바일 터치 최적화

  [2] 결과 카드 생성 및 SNS 공유
      - 워크숍 완료 후 시각적 결과 카드 자동 생성
      - 카카오톡, 인스타그램 스토리, 트위터(X) 공유 버튼
      - Open Graph 메타 태그 최적화로 미리보기 자동 생성
      - 공유 시 "나도 해보기" 링크 포함 → 바이럴 유도

  [3] 개인 기록 (선택적 회원가입)
      - 비회원도 체험 가능 (핵심 가치: 접근성)
      - 회원가입 시 워크숍 이력·기록 저장
      - 감정 변화 추이 시각화 (주간/월간 리포트)

  [4] 큐레이션 시스템
      - 오늘의 추천 워크숍
      - 기분·상황별 워크숍 추천
      - 시리즈형 워크숍 (예: "자존감 7일 챌린지")

  [5] 경량 운영 설계
      - 서버·DB 없이 정적 HTML만으로 운영
      - 콘텐츠 추가 = HTML 파일 1개 추가 (템플릿 복사)
      - 유지보수 업무 최소화 → 콘텐츠 제작에 집중

      - 워크숍 기반 오프라인 프로그램 운영

8. 기술 스택 및 개발 계획
────────────────────────────────────────────────────────────────

■ 개발 체제
  - 1인 개발 (기획·디자인·개발·운영 겸임)
  - 유지보수 최소화를 위한 정적 HTML 중심 구조

■ 기술 구성

  구분           기술
  ─────────────────────────────────────────────
  핵심 구조      순수 HTML + CSS + Vanilla JS
                 (프레임워크 미사용 → 빌드 불필요)
  인터랙션       CSS 애니메이션 + 경량 JS
                 (카드 전환, 슬라이더, 입력 폼)
  결과 카드      html2canvas 또는 Canvas API
                 (사용자 결과를 이미지로 변환)
  호스팅         GitHub Pages (무료, 무서버)
                 또는 Netlify / Vercel 정적 호스팅
  SNS 공유       Web Share API + 카카오 JS SDK
                 Open Graph 메타 태그
  데이터 저장    localStorage (개인 기록)
                 서버·DB 없음 → 유지보수 제로

  ★ 기술 선택 근거
  - 서버 없음 → 서버 비용 0원, 장애 가능성 최소
  - 빌드 없음 → HTML 파일 수정만으로 콘텐츠 추가
  - 1인 운영 → 복잡한 인프라 배제, 콘텐츠에 집중
  - 정적 사이트 → 트래픽 급증에도 안정적 (CDN)

■ 사이트 구조 (파일 기반)

  /index.html                    ← 메인 페이지 (워크숍 목록)
  /workshop/감정날씨.html        ← 개별 워크숍 페이지
  /workshop/강점발견.html        ← 개별 워크숍 페이지
  /workshop/...                  ← 워크숍 추가 = HTML 파일 추가
  /css/style.css                 ← 공통 스타일
  /js/workshop.js                ← 공통 인터랙션 로직
  /js/share.js                   ← SNS 공유 기능
  /img/                          ← 일러스트·아이콘

  * 워크숍 추가 시 HTML 파일 1개만 작성하면 완료
  * 템플릿 HTML을 복사 → 내용만 교체하는 방식

■ 개발 로드맵 (1인 기준)

  Phase 1 (2~3주) — MVP 출시
  ├── 공통 템플릿 HTML/CSS 제작
  ├── 워크숍 인터랙션 JS 모듈 개발
  ├── 핵심 워크숍 3개 제작
  ├── 결과 카드 이미지 생성 + SNS 공유
  └── GitHub Pages 배포

  Phase 2 (1~2개월) — 콘텐츠 확장
  ├── 워크숍 10개로 확장 (템플릿 복사 방식)
  ├── localStorage 기반 개인 기록 기능
  ├── 메인 페이지 카테고리별 필터링
  └── SEO 및 Open Graph 최적화

  Phase 3 (3~4개월) — 안정화 및 확산
  ├── 워크숍 20개 이상 확보
  ├── 시리즈형 워크숍 (챌린지) 구성
  ├── 사용자 피드백 수집 (구글 폼 연동)
  └── 기관 제휴 및 홍보 활동

9. 운영 체제 (1인 개발)
────────────────────────────────────────────────────────────────

■ 1인 운영자 역할

  영역                  업무 내용
  ─────────────────────────────────────────────
  기획                  워크숍 주제 선정, 구조 설계
  개발                  HTML/CSS/JS 페이지 제작
  디자인                Canva·Figma 활용 간편 디자인
  콘텐츠                심리 워크숍 시나리오 집필
  운영                  배포, SNS 운영, 피드백 관리

■ 1인 운영 효율화 전략
  - 템플릿 기반 제작 → 워크숍 1개 추가에 1~2시간
  - 서버 없음 → 인프라 관리 업무 제로
  - GitHub Pages 자동 배포 → push만 하면 반영
  - 심리 전문가 자문은 외부 협력으로 확보
    (대학 심리학과, 상담센터 등 재능기부·MOU)
