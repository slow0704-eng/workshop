2. 즉시 대응 (높★) — 코드 반영 5건
────────────────────────────────────────────────────────────────

  ■ EX-04 이미지 저장 버튼 연타 방지

  대응: saveResultImage() 시작 시 btn.disabled = true
       완료/에러 후 복원
  반영: result.js

  ■ EX-14 Clipboard API 실패 폴백

  대응: .catch() 추가 → prompt('링크를 복사해주세요:', url)
  반영: 모든 워크숍 HTML의 인라인 onclick

  ■ EX-15 SNS 공유 URL 한글 인코딩

  대응: 공유 URL을 encodeURIComponent()로 인코딩
  반영: 모든 워크숍 HTML의 트위터/페이스북 href

  ■ EX-18 iOS Safari 다운로드 대응

  대응: iOS 감지 → canvas.toBlob() → 새 탭에서 이미지 열기
       "길게 눌러 저장하세요" 안내
  반영: result.js saveResultImage()

  ■ EX-28 강점발견 0개 체크 모순 메시지

  대응: showResult()에서 checkbox 0개 체크 감지 시
       "아직 발견하지 못해도 괜찮아" 대안 카드 표시
  반영: result.js + workshop/강점발견.html

3. v2 추가 대안 흐름 (UC별)
────────────────────────────────────────────────────────────────

  ■ UC-U04 미니 워크숍 체험 — 추가 대안 흐름

  A3. 브라우저 뒤로가기로 이전 단계 복귀 (EX-01)
      1. 사용자가 Q2에서 뒤로가기
      2. URL #q2 → #q1으로 변경
      3. Q1 화면 다시 표시, 이전 radio 선택 유지
      4. 재선택 가능, "다음"으로 다시 진행 가능

  A4. 새로고침으로 진행 상태 유실 (EX-02)
      1. 사용자가 Q3에서 F5 새로고침
      2. URL #q3 유지 → Q3 화면 표시
      3. 모든 radio 선택 초기화됨
      4. 사용자가 다시 선택하여 진행 가능
      5. ★ 처음부터 하고 싶으면 헤더 "홈" 탭

  A5. 이미지 저장 버튼 연타 (EX-04)
      1. 사용자가 "이미지 저장" 빠르게 2회 탭
      2. 첫 번째 클릭: btn.disabled = true → 저장 진행
      3. 두 번째 클릭: disabled 상태라 무시됨
      4. 저장 완료 2.5초 후 btn.disabled = false 복원

  ■ UC-U07 결과 카드 확인 — 추가 대안 흐름

  A2. 미선택 시 "모르겠어" 자동 처리 (v1에서 추가, 유지)

  A3. iOS Safari에서 이미지 다운로드 실패 (EX-18)
      1. 시스템이 iOS 환경을 감지
      2. Canvas → Blob → 새 탭에서 이미지 열기
      3. 캐릭터 대사: "이미지를 길게 눌러서 저장해봐!"

  A4. 강점발견 0개 체크 시 모순 메시지 (EX-28)
      1. 사용자가 체크 0개로 "결과 보기"
      2. showResult()가 체크 0개 감지
      3. "체크한 것이 강점" 대신 대안 카드 표시:
         "아직 잘 모르겠어도 괜찮아.
          강점은 천천히 발견할 수 있어."

  ■ UC-U09 결과 SNS 공유 — 추가 대안 흐름

  A2. Clipboard API 실패 (EX-14)
      1. 사용자가 "링크 복사" 탭
      2. navigator.clipboard 미지원 또는 HTTP 환경
      3. .catch() 발동 → prompt 대화상자에 URL 표시
      4. 사용자가 수동 복사

  A3. SNS 공유 URL 한글 인코딩 (EX-15)
      → 공유 URL이 encodeURIComponent()로 인코딩되어 전달
      → 트위터/페이스북이 정상 파싱

  ■ UC-U16 위기 상담 — 추가 대안 흐름

  A1. PC에서 tel: 링크 무반응 (EX-26)
      1. PC 사용자가 전화번호 클릭
      2. 프로토콜 핸들러 없음 → 무반응 또는 에러
      3. 전화번호가 텍스트로도 보이므로 직접 전화 가능
      4. ★ 온라인 상담 링크 병기 검토 (Phase 2)

  ■ 전체 UC 공통 — JS 비활성 시 (EX-06)

  <noscript> 태그로 안내 표시:
  "이 워크숍은 일부 기능(결과 이미지 저장)에
   JavaScript가 필요해요.
   브라우저 설정에서 JavaScript를 활성화해주세요."
